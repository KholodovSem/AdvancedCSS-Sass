CSS Architecture

-----------------------------Design-----------------------

Component-driven design:
- Modular building blocks that make up interfaces;
- Held together by the layout of the page;
- Re-usable across a project, and between different projects;
- Independent, allowing us to use them anywhere on the page.


Atomic design:

Atoms => Molecules => Organisms => Templates => Page

Атом - самый маленький элемент страницы.

Молекула - соеденяясь атомы образуют компоненты.

Организм - Компоненты в свою очередь соеденяясь образуют модули.

Шаблоны - Компоненты объденяясь образуют Шаблоны

Страница - Ну а страница состоит из шаблонов.

-----------------------------Build-----------------------

BEM (Block Element Modifier) - метод именования элементов.

Block - автономный компонент, который имеет смысл сам по себе.

Element - элемент блока который сам по себе не имеет никакого смысла.

Modifier - флаг, который мы можем поместить на элемент, чтобы сделать его отличным
от обычной версии блока или элемента.

-----------------------------Architecture-----------------------

The 7-1 Pattern - заключается в том, что у нас есть семь разных папок для 
частей Sass, и один главный Sass-файл, куда мы испортируем эти частицы.

The 7 Folders:

base/
- Базовые стили

components/
- Файлы для каждого отдельного компонента

layout/
- Папка с макетами

pages/
- Папка для страниц, в котой мы храним стили для каждой отдельной страницы

themes/
- Если мы хотим реализовать различные визуальные темы

abstracts/
- Переменные, миксины. То что не выводит css.

vendors/
- Сторонний CSS
